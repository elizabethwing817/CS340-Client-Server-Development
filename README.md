# CS 340 Client-Server Development Portfolio Reflection

## How do you write programs that are maintainable, readable, and adaptable?

Throughout this course, I learned the importance of writing code that is organized, modular, and easy to maintain. In Project One, I created a reusable CRUD Python module that handled the database operations for creating, reading, updating, and deleting records in MongoDB. By separating these database functions from the dashboard application, I was able to reuse the same module in Project Two without rewriting database code. This made the dashboard easier to develop, easier to debug, and more adaptable to future changes. In the future, this CRUD module could be reused in other applications that require MongoDB database operations, reducing development time and improving code organization.

## How do you approach a problem as a computer scientist?

When developing the dashboard for Grazioso Salvare, I approached the project by breaking the requirements into smaller tasks and completing one feature at a time. I first ensured that the database connection and CRUD functionality worked correctly before integrating them into the dashboard. After that, I implemented the filtering options, data table, charts, and map individually while testing each feature before moving on to the next. This project differed from previous programming assignments because it required integrating multiple technologies, including MongoDB, Python, Dash, and Jupyter Notebook. In future projects, I will continue using an incremental development approach by testing each component individually before combining them into a complete application.

## What do computer scientists do, and why does it matter?

Computer scientists develop software solutions that help organizations organize, analyze, and use data more effectively. In this project, the dashboard provided Grazioso Salvare with an easier way to search and visualize animal shelter data using interactive filters, charts, and maps. Rather than manually searching through large collections of records, users can quickly locate the information they need and make better decisions. This type of software improves efficiency, reduces the time required to analyze data, and allows organizations to focus on accomplishing their goals more effectively.
